General CSS rules

  • Use class based selectors whenever possible
  • Do not ever use id selectors
  • Do not ever use !important
  • Bind javascript events to selectors prefexed: .js-*

File organization

  • One BEM block = one preprocessor file.
  • The file with the styling of the BEM block should be named the same as the block itself.

Nesting of selectors

  • The less nesting levels, the better.
  • Do not allow more than 3 levels of nesting (pseudo-elements, pseudo-selectors and media-conditions are not considered to increase nesting).
  • Always add an empty line before the nested selector or @media.
  • Always make an additional indentation for nested selectors.
            
            .promo {
              display: block;
                                // empty line for better readability
              &__link {         // first level of nesting
                color: #ff0000;
                                // empty line for better readability
                &:hover {       // doesn't increase the level of nesting
                  ...
                }
              }
            }

Ampersand

  • Use the ampersand only before: separator of the BEM element, separator of the BEM modifier, a pseudo-element or a pseudo-selector.
  • Never use an ampersand at the places of separation of word combinations of block names, elements or modifiers (see example).
  • Never repeat an ampersand-written selector within one context.
            
            .promo {
        
              &:hover {     // Right: Ampersand before pseudo-class
                ...
              }
        
              &__img {      // Right: Ampersand before of the element delimiter
                ...
              }
        
              &__item {
                &-link {    // Wrong: Ampersand in the place of word separation in the name of the element
                ...
                }
              }
        
              &-shower {    // Wrong: Ampersand in the place of wording in the name of the block
                ...
              }
        
              &--large {    // Right: modifiers should be written below the elements
                ...
              }
            }

@media

  • Insert @media to the selectors, and not vice versa.
  • Don't put @media into each other.
  • Prefer the path of mobile-first, avoid specifying the @media-condition max-width in favor of min-width.
  • Write @media near each other, don't write selectors between them.
  • Write the conditions for @media where you use @media, do not write conditions in variables. 
            
            .promo {
              display: block;
        
              // Right: the condition is clear, in the variable only the width
              @media (min-width: $screen-md) {
                display: none;
        
                &__img {
                  ...
                }
              }
        
              @media (min-width: $screen-lg) {
                display: block;
              }
        
              // Wrong: the condition is entirely put into a variable (non-obviousness)
              @media ($mobile-width) {
                display: block;
              }
            }

Variables

  • Add to variables colors (more about colors in variables), font-sizes, font-families, breakpoints etc.
  • Separate the semantic blocks with variable empty lines and comments.
            
            //----------------------------
            // Color pallet
            //----------------------------
        
            // Gray
            $gray33:                      #333;
            $gray46:                      #464646;
        
            // Red
            $red:                         #ff0000;
        
            // Green
            $green:                       #2ECC40;
        
            // Blue
            $blue:                        #0074D9;
        
        
            //----------------------------
            // Color variables
            //----------------------------
            $color-main:                  $blue;
            $color-success:               $green;
            $color-danger:                $red;
        
            //----------------------------
            // Font
            //----------------------------
        
            // Font-size
            $font-size-xs:                1.2rem;
            $font-size-sm:                1.4rem;
            $font-size-md:                1.6rem;
            $font-size-lg:                1.8rem;
            $font-size-xlg:               2rem;
        
            // Font-family
            $font-family:                 'Roboto', Arial, sans-serif;
            $font-family--secondary:      'Helvetica', Arial, sans-serif;
        
            //----------------------------
            // Breakpoints
            //----------------------------
            $screen-xs:                   320px;
            $screen-sm:                   768px;
            $screen-md:                   1170px;
            $screen-lg:                   1366px;

Sass placeholders

  • Placeholder selectors will not show up in the generated CSS, only the selectors that extend them will be included in the output. You can create separate file with placeholders and called it helpers and using this helpers in the project. For better understanding read this article - Understanding placeholder selectors 
            
            %list-reset {
              transition: background-color ease .2s;
              margin: 0 .5em;
            }
        
            .product-list {
              @extend %list-reset;
              /* error specific styles... */
            }
        
            .user-list {
              @extend %list-reset;
              /* info specific styles... */
            }
