Why BEM?

  • Self-documenting.
  • Ease of code maintenance.
  • Reusable blocks in project.
            
            <div class="bem-block">
                <div class="bem-block__el bem-block__el--mod">
                    <span class="name">Item 1</span>
                </div>
        
                <div class="bem-block__el bem-block__el--mod">
                    <span class="name">Item 2</span>
                </div>
        
                <div class="bem-block__el bem-block__el--mod">
                    <span class="name">Item 3</span>
                </div>
            </div>

Block is a separate part of the page

  • The class name should be simple and short.
            
            <!-- .product — BEM-block -->
            <div class="product">
                ...
            </div>

BEM blocks can and should be nested in each other

  • You can nested block inside another block.
            
            <!-- .page-header — BEM-block -->
            <div class="page-header">
        
                <!-- .logo — nested BEM-block -->
                <a class="logo">...</a>
            </div>

Element is part of the BEM block

  • The name of the class is formed from the name of the block with the addition of __ and the name of the element.
  • The class name should be simple and short.
            
            <div class="product">
                <!-- .product__image — BEM-element of product block -->
                <img class="product__image" src="..." alt="...">
        
                <!-- .product__description — BEM-element of product block -->
                <p class="product__description">...</p>
        
                <!-- .product__more-link — BEM-element of product block -->
                <a class="product__more-link" href="">...</a>
            </div>

How to differentiate a BEM-block from a BEM-element

  • Just ask yourself the question: "This code part may be required to me separately, in itself? Or is it only necessary inside her parent?" If you need it separately, this is a BEM-block, if it is conceivable only inside the parent - it's a BEM-element.
  • In really doubtful cases, make a choice in favor of BEM-block.
  • Do not forget about mixing (it is possible to have at the same level a BEM-level class of some parent block, and its BEM-block level class).

Modifier - an additional class for changing the design or behavior

  • The name of the class is formed from the name of the block / element with the addition - and the name of the modifier.
  • The name should be simple and short.
  • The class name can answer the questions "What is this?", "What changes?", "What is different from the others?" 
            
                <!-- .product--large — BEM-modifier -->
                <div class="product  product--large">
                  ...
                </div>

The modifier can't be used single

  • The modifier class should never be used single, but always only with the class that it modifies. 
            
                <!-- Right usage -->
                <div class="product  product--large">
                  ...
                </div>
        
                <!-- Wrong usage -->
                <div class="product--sale">
                  ...
                </div>

Mixing - combining on one tag classes BEM-block and BEM-element

The combination is possible in any combination: BEM-block + BEM-element, BEM-block + BEM-block, BEM-element + BEM-element.

  • Classes of BEM blocks should be written first.
  • Add some style properties needed only at the location of the addition (using the modifier is irrational). Example: for .btn inside .page-header, an external left margin of 37 pixels is required. You can add an additional class .page-header__btn for the .btn tag and indent with this selector. This is a normal practice, you can safely use it.
            
                <!-- Example-1 -->
                <div class="page-header">
                    <div class="page-header__nav-wrap">...</div>
                    <a class="btn  page-header__btn" href="/buy">...</a>
                </div>
        
                <!-- Example-2 -->
                <div class="slider">
                  <div class="promo  slider__inner">
                    <div class="promo__item  slider__item">
                      ...
                    </div>
                  </div>
                </div>
